A great plan for a major challenge

Nowadays, media content creation is a very important part of the digital world. Everybody on the planet should be able to express himself through film making.

But it's not only that, what we want to build is a professional grade video editing application, and this is what this project is about: build upon the currently well settled codebase to reach professional standards !

The plan: bring you what you always wanted from a video editor

The items on our roadmap are based on our own experience with filmmaking and the results of our user survey you are of course very welcome to provide (constructive) feedback on our roadmap!

Work package 1: Stabilization to reach the long awaited 1.0 release!

Quality Assurance

Perfect our Quality Assurance infrastructure so we can keep building on top of our codebase and avoid regressions ! At this stage we will also make sure to provide daily build packages in repositories for major Linux distibutions (Fedora, RHEL, Debian and Ubuntu).

Massive bug fixing

Even though we have been working hard for the lasts months (years?) to fix bugs all around the multimedia stack, more work is still needed to reach the level of stability we are targetting.

Documentation

Provide user documentation as well as developer documentation of good quality to make sure anyone who wants to help can get started fast.

Work package 2: Implement most wanted features this is your choice!

Hardware accelerated decoding and encoding

In GStreamer 1.2 all the infrastructure that will allow us to cleanly take advantages of the video decoding capacity of your graphic card is there. We now need to make sure that our use case is supported with the most common graphic cards (intel trough VAAPI and nvidia through VDPAU)

Speed control

Letting you change the speed rate of your clips is something that we already started to work on, we want to make sure that it works flawlessly, and we want to provide an intuitive interface in Pitivi.

Semi advanced title creation

Current title management is pretty basic, in that work package we will make sure that all the main features are implemented and rock-solid. We will also make it possible to animate and fine tune titles with a nice User Interface.

Color correction

Great control over color is definitely a must have feature for a professional grade video editing application. Currently it is possible to workaround our lack of proper interface for that, but we really want to create the needed tools so that you can fine tune your videos using state of the art colorimetry techniques

Proxy editing

Proxy editing is the ability to swap clips by a "proxy" version that is more suited for editing, and then using the original, full-quality clip to do the render. This is useful when working with input formats that are not efficient/fast enough for live editing or when dealing with high definition/computationally expensive footage for which the computer is not powerful enough to handle realtime playback. This is also something that has already been tackled but that still needs quite a bit of work to reach the necessary level of maturity.

EDL support

EDL is the simplest and most used project format, it's a standard that allows to communicate with other video edition and post production softwares. We need to implement its support right into the GStreamer Editing Services.

Windows

Port pitivi to Windows and provide installer for 1- daily builds 2- stable versions

Mac OS

Port pitivi to Mac OS X and provide installer for 1- daily builds 2- stable versions

Those various features are not inter dependant, and you can vote to choose what you would like to see implemented.
