A big plan: for a major problem

Nowadays, media content creation is a very important part of the digital world, everybody on the planet should be able to express himself through film making.

But this is not it, what we want to achieve is to build a professional grade video editing application. And this is what this whole project is actuall about: build uppon the currently well setted codebase to reach professional standard!

The plan: bring you what you always wanted from a video editor

The items on our roadmap are based on our own experience with filmmaking and the results of our user survey you are of course very welcome to provide (constructive) feedback on our roadmap!

Work package 1: Stabilization to reach the long awaited 1.0 release!

Quality Assurance

Perfectionnate our Quality Assurance infrastructure so we can keep building on top of our codebase avoiding regressions!

Massive bug fixing

Even though we have been working hard for the lasts monthes (years?) to fix bugs all around the multimedia stack, much more work is needed to arrive to the level of stability we are targetting.

Documentation

Provide user documentation as well as developer documentation of good quality to make sure anyone who want to help can get started fast.

Work package 2: Implement most wanted feature and opimize the current codebase

Hardware accelerated decoding

In GStreamer 1.2 all the infrastcture that will allow us to cleanly take advantages of the video decoding capacity of your grafic card is there. We now need to make sure that our use case is supported with most used grafic cards (intel trough VAAPI and nvidia through VDPAU)

Speed control

Letting you change the speed rate of your clips is something that we already started to work on, we want to make sure that it flowlessly works and we want to provide a simple to use Interface in Pitivi.

Semi advanced title creation

Current title management is pretty basic, in that work package we will make sure that all main needed features are implemented and solid. We will also make it possible to animate and fine tune titles with a nice User Interface.

Colorimetry

Great control over color is definitely a must have feature for a professional grade video editing application. Currently it is possible to workaround our lack of proper interface for that, but we really want to create the needed tools so that you can fine tunes your videos using state of the art colorimetry techniques

Proxy editing

Proxy editing is the ability to swap clips by a "proxy" version that is more suited for editing, and then using the original, full-quality clip to do the render. This is useful when working with input formats that are not efficient/fast for live editing or when dealing with high definition/computationally expensive footage for which the computer is not powerful-enough to handle realtime playback. This is also something that as already been tackled but that still needs quite a bit of work to reach the necessary level of maturity.

EDL support

EDL is the simplest and most use project formatting format to be able to communicate with other video edition and poost production softwares. We need to implement its support right into the GStreamer Editing Services.
